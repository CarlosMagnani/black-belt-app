# BlackBelt

Management platform for Brazilian Jiu-Jitsu academies. Connects owners, professors, and students around class attendance, belt progression, and membership.

## Language

**Academy**:
A martial arts school registered on the platform. Created by an owner, joined by students via invite code.
_Avoid_: School, dojo, gym

**Owner**:
The user who created and manages an academy. One owner per academy, one academy per owner.
_Avoid_: Dono, admin, manager

**Professor**:
An academy role granted to an existing student by the owner. Professors retain student membership and rank history, and may validate check-ins only for classes they teach; they cannot change student belts or degrees and are not a separately onboarded account type.
_Avoid_: Instrutor, coach, teacher

**Student**:
An academy member who trains and attends classes. Joins via invite code. Cannot self-assign belt or approve check-ins.
_Avoid_: Aluno, member, trainee

**Invite Code**:
A unique code in the format `BB-XXXXXX` generated by the system when an academy is created. Required for any student to join an academy.
_Avoid_: Join code, access code, registration code

**Belt**:
A student's rank in the BJJ hierarchy: white, blue, purple, brown, black, coral, red. Controlled exclusively by the academy owner after initial declaration.
_Avoid_: Rank, grade, level

**Degree**:
A stripe on a belt, numbered 0-4. A student becomes eligible for promotion after 24 approved check-ins at their current level.
_Avoid_: Stripe, grau, rank

**Promotion Eligibility**:
The state reached after 24 approved check-ins at a student's current belt and degree. It permits, but never automatically performs, a rank change.
_Avoid_: Automatic promotion, guaranteed promotion

**Check-in**:
A student's attendance request for a specific class. Always starts as pending; requires owner or professor approval to count toward belt progression.
_Avoid_: Attendance, presence, presença

**Recurring Class**:
A weekly class definition owned by an academy, identified by day, start time, duration, instructor, location, and level. It is not an individual class session.
_Avoid_: Class event, session, one-off class

**Membership Plan**:
A payment option defined by the academy owner (name, price, period). Does not process payments in MVP.
_Avoid_: Subscription, package, plano

**Membership Subscription**:
A student's link to a specific membership plan within an academy. Status is managed manually by the owner.
_Avoid_: Membership, plano do aluno

**Roster**:
The academy's operational list of students and professors. It is the owner's single place to manage people, including professor roles, rank progression, and subscription assignment.
_Avoid_: Student list, member list, contacts

**Average Attendance**:
The mean number of approved check-ins per scheduled class occurrence in the preceding 28 days, including occurrences with no approved check-ins.
_Avoid_: Average occupancy, utilization

**Activity Event**:
An immutable audit record of a meaningful academy operation, identifying the academy, actor, action, affected entity, and time. It supplies the owner activity feed; it is not an asynchronous domain-event mechanism.
_Avoid_: Event bus message, notification, log line
